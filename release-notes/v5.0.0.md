# Release Notes for vsync v5.0.0

## Release Date

2026-03-02

## Overview

This major release introduces hardware-based vblank timestamping for improved accuracy, a new framesync demo application with live monitoring, Intel Arc Battlemage platform support, and build system modernization with Meson and CMake.
We recommend that all users upgrade to this version as it contains significant improvements in synchronization accuracy, build systems, and platform support.

## What's New

* Hardware-based vblank timestamping for improved synchronization accuracy
* New framesync demo application for visual testing
* Live graph for real-time sync status monitoring and drift visualization
* Intel Arc Battlemage (BMG/B-Series) platform support
* Introduced Meson and CMake build systems and phased out Make
* Pipelock mode for in-process primary-secondary sync
* Multi-pipe support in single secondary process
* Multi-client support with background vsync collection
* Iterative refresh period adjustment in test applications
* Shared DPLL patch support up to kernel 6.19
* Added clang-format and clang-tidy configuration files for code consistency

## Bug Fixes/Enhancements

* Applications renamed for better clarity (swgenlock, vblmon, pllctl)
* Documentation refactored into separate modular files
* DRM patch check now skipped for Hammock Harbor mode
* Python plotter updated with average time metrics
* Kernel version requirements documented for non-Hammock Harbor mode

## Known Issues

* Screen flicker: Value for SHIFT may be too large and causes flickering on screen when synchronized, adjust the value to resolve.

## Breaking Changes

* **API Change**: `vsync_lib_init` function signature has been modified. Code using v4.x API will need to be updated.
* Make build system has been phased out in favor of CMake and Meson.
* This version is **not** backward compatible with 4.x releases due to API changes.
